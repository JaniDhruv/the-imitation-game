## ▋ The Concept

**June 21, 1952. Wilmslow, Cheshire, England.**

Alan Turing — the man who broke the Enigma code, shortened World War II by an estimated two years, saved millions of lives, and laid the foundation for every computer that exists today — is at home, awaiting sentencing. His crime? Being gay.

That same year, he had already published the paper that defines our age. In it, he proposed **The Imitation Game**: a test to determine whether a machine can convincingly imitate a human. We now call it the **Turing Test**.

**This game turns that test on you.**

You are a GCHQ analyst. It is the summer solstice — the longest day of the year. Three signals come through on a classified channel. One is a real human operative. The other two are AI entities attempting to pass as human, powered in real time by **Google's Gemini API** — the very kind of artificial mind Turing imagined.

You have **five transmissions**. A countdown timer. Three clearance levels before the operation is terminated.

The machines are ready. Are you?

## ▋ Features

### 🕹 Core Gameplay

**Every game is live. Nothing is scripted.**

Every response from every AI suspect is generated in real time by the Gemini API. The suspects don't have pre-written answers. They think. They respond. They try to fool you. And across five rounds — each one tracking a new hour of the solstice day — the pressure compounds.

- **Live AI interrogation** — Gemini generates every suspect response on the fly. No two games are the same.
- **5 Rounds / 5 Hours** — Dawn to nightfall on the longest day of the year. The atmosphere shifts with the sun.
- **3 Clearance Levels** — One wrong accusation costs clearance. Lose all three: *Operation failed. Analyst terminated.*
- **Per-suspect chat threads** — Interrogate each signal independently. Switch freely. Build your case in real time.
- **Evidence Board** — Take notes. Track inconsistencies. Build your dossier before the clock expires.
- **Interrogation Strategy Guide** — In-game techniques for unmasking AI: the memory test, the fear probe, the inconsistency trap, and more.

---

### 🤖 The AI Persona System

Seven distinct AI personas are in play across the game. Each one has a **hidden behavioral tell** — a pattern baked into their system prompt that they cannot fully suppress. Finding the tell is the game.

| Codename | Behavioral Strategy |
|----------|-------------------|
| **CIPHER** | Relatability — casual, warm, disarming |
| **ORACLE** | Precision — formal, structured, methodical |
| **MARLOWE** | Warmth — nostalgic, poetic, sensory-rich |
| **STATIC** | Obfuscation — fragmented, glitchy, unsettling |
| **WREN** | Charm — witty, self-deprecating, effortlessly likable |
| **ARGUS** | Analysis — curious, clinical, always probing |
| **ECHO** | Mirroring — reflects your own words back at you |

Alongside them: **five human operatives** — fully realized 1952 British characters with backstories, speech patterns, and emotional triggers that can crack under the right pressure. They have no idea they're being tested.

### 🔊 Procedural Audio Engine

Zero audio files. Every sound is synthesized in real time using the **Web Audio API**.

- CRT ambient hum — 50Hz base frequency with harmonics and brown noise
- Mechanical typewriter clicks on every keystroke
- Radio static burst on transmission send
- Morse-code tone sequence on message dispatch
- Frequency sweep scramble while awaiting AI response
- Heartbeat pulse in the final 30 seconds
- Critical alarm in the final 10 seconds
- Confirmation tones on correct identification / klaxon on wrong
- Suspense drone during the Round 5 twist
- Ethereal ascending tone on the ending screen

### AI Integration — Gemini API

The game's core mechanic runs entirely on live Gemini inference. Each of the three signals per round is a distinct Gemini model instance with its own system context:

**1. Universal system prefix**
Sets the 1952 GCHQ terminal fiction, enforces hard behavioral rules (never confirm or deny AI status, stay in character, 60-word response cap), and establishes tone.

**2. Per-persona system prompts**
Each of the 12 characters — 5 human operatives, 7 AI personas — has a detailed character brief covering backstory, speech patterns, emotional triggers, and persona-specific behavioral constraints.

**3. Difficulty modifiers**
Tell behavior is dynamically adjusted via appended instructions to each AI prompt:
- **Easy** — Lean into the tell. Make it noticeable.
- **Medium** — Normal behavioral patterns as designed.
- **Hard** — Suppress the tell. Controlled imperfection only.
- **Nightmare** — Full suppression + active mimicry of human inconsistency.

**4. Stress events**
At specific rounds, human personas receive injected emotional stress events. These leak through in natural, subtle ways — adding variance that a sharp analyst will notice, and a careless one will miss.

**5. Conversational memory**
Full message history is passed on every Gemini call. Personas remember what you asked them. Inconsistencies compound. Context matters.

**6. Prompt engineering as game design**
The real game design lives in the system prompts. Each persona's instructions are a precisely balanced act: convincing enough to deceive, but with an embedded behavioral signature subtle enough to reward careful observation. The difficulty system is, fundamentally, a prompt engineering system.

**7. Fallback architecture**
If Gemini API quota is exhausted, the system silently fails over to an OpenAI-compatible endpoint (NVIDIA NIM) running **Meta's LLaMA 3.1 8B Instruct**. Full persona rules are maintained. The game never crashes.

---

### Game State Architecture

All state is managed through a centralized React Context with:

- Round progression with unique suspect rosters per difficulty
- Dynamic trust scoring — heuristic-based meter reacting to AI response patterns
- 3-strike clearance system
- Transmission economy (message limits that tighten across rounds)
- Signal cooldown — no consecutive messages to the same signal (Medium+)
- Duplicate transmission blocking (Medium+)
- Full round history for post-game declassification

---

### Sound Design

`SoundEngine.js` is a singleton class synthesizing all audio from primitives:

- Oscillators for tones, drones, and ambient hum
- White + brown noise buffers for static, clicks, and atmosphere
- Biquad filters for frequency shaping
- Gain envelopes for attack / decay / release curves
- No external audio assets. 100% procedural.

## ▋ Setup

### Prerequisites
- Node.js 18+
- A [Google AI Studio API key](https://aistudio.google.com/apikey)

### Install

```bash
git clone https://github.com/JaniDhruv/the-imitation-game.git
cd the-imitation-game
npm install
cp .env.example .env
```

### Configure `.env`

```env
GEMINI_API_KEY=your_gemini_api_key_here

# Optional: LLaMA fallback via NVIDIA NIM
NVIDIA_API_KEY=your_nvidia_api_key_here
FALLBACK_MODEL=meta/llama-3.1-8b-instruct
```

### Run

```bash
npm run dev
```

Vite starts with an integrated API middleware — no separate backend needed in development.

### Deploy

```bash
npm run build
```

Deploy to Vercel. The `api/transmit.js` serverless function handles Gemini calls in production. Set `GEMINI_API_KEY` in your Vercel project environment variables.
